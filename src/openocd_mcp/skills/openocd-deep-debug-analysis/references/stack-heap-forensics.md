# 栈堆内存取证

## 1. 目标 (Objective)
用于定性栈溢出、堆损坏、越界写入和内存踩踏，帮助缩小污染范围与嫌疑对象。

## 2. 输入 (Input)
- 栈顶、栈底、堆起止地址
- 内存分配器元数据结构信息（若可得）
- 可疑地址或崩溃地址

## 3. 步骤 (Steps)
1. 检查栈哨兵、watermark 或保护字是否被改写。
2. 对可疑栈帧附近做窗口化导出，识别重复覆写模式。
3. 检查堆块头、空闲链、边界标记是否完整。
4. 将污染区域与任务栈、DMA 缓冲或外设缓存做交叉比对。

## 4. 输出 (Output)
- 污染范围
- 可疑数据模式
- 更可能的破坏来源候选

## 5. 风险 (Risks)
- 盲读非法内存会触发新异常。
- 现场已被后续写入覆盖时，污染源可能已消失。
- 若堆实现未知，强行解释 Header 会误导判断。

## 6. 示例 (Example)
```text
stack_guard @ 0x20001FFC = 0x00000000
heap_block[3].next = 0xFFFFFFFF
初判：任务栈越界写入破坏了邻近堆元数据
```
