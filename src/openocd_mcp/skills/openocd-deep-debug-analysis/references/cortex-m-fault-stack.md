# Cortex-M 异常栈解析参考

## 1. 目标 (Objective)
指导如何手动分析 Cortex-M 异常压栈，恢复异常发生前的寄存器现场与执行点。

## 2. 输入 (Input)
- 异常时的 SP
- SP 周边至少 32 字节内存
- 当前使用的是 MSP 还是 PSP 的线索

## 3. 步骤 (Steps)
1. 判断当前异常使用的是 MSP 还是 PSP。
2. 依顺序提取 `R0 R1 R2 R3 R12 LR PC xPSR`。
3. 用压栈 PC 与 LR 对照当前寄存器，判断停机点与返回路径。
4. 若涉及 FPU 扩展栈帧，明确说明额外上下文需求。

## 4. 输出 (Output)
- 异常前寄存器还原表
- 压栈来源说明
- 对 PC/LR 的简要解释

## 5. 风险 (Risks)
- 栈溢出会导致压栈数据被污染。
- 中断嵌套会使当前 SP 不等于首次异常入口 SP。
- 误把当前 PC 当成压栈 PC 会导致错误定位。

## 6. 示例 (Example)
```text
SP=0x20001000
stacked_pc=0x08003456
stacked_lr=0x08001221
结论：异常发生前最后一条有效返回链落在 foo_bar() 附近
```
