# 典型外设映射模型参考 (STM32)

## 1. 目标 (Objective)
指导如何在 STM32 架构上快速关联 RCC、GPIO、UART、I2C 等外设配置，形成统一排查思路。

## 2. 输入 (Input)
- STM32 具体家族或系列
- 外设基地址或 SVD 节点
- 关联时钟树与 GPIO 端口信息

## 3. 步骤 (Steps)
1. 先确认芯片家族，避免混用不同系列寄存器布局。
2. 通过基地址和偏移量定位 RCC、GPIO、外设寄存器。
3. 用位掩码解释启停、错误、复用、模式等关键字段。
4. 将时钟、GPIO、外设本体三层状态串起来看，而不是只读单寄存器。

## 4. 输出 (Output)
- STM32 专属寄存器译码结果
- 所处层级的问题定位
- 与系列差异相关的注意事项

## 5. 风险 (Risks)
- 不同 STM32 族群寄存器布局差异很大。
- 把一个系列的位定义套到另一系列会得到完全错误的结论。
- 只看外设本体而忽略时钟树与 GPIO 复用会误判为“硬件损坏”。

## 6. 示例 (Example)
```text
芯片：STM32H7
GPIOA.MODER[5:4] = 10b
GPIOA.AFRL[7:4]  = 0111b
结论：PA2 已配置为复用功能，需继续检查 USART2 时钟与 UE 位
```
