# 安全门禁与连接规则

## 1. 目标 (Objective)
定义 OpenOCD 连接与接管阶段必须遵守的安全边界，确保“先确认边界，再获取控制”。

## 2. 输入 (Input)
- 目标板基本信息：接口类型、供电方式、芯片家族
- OpenOCD 配置：`interface`、`transport`、目标脚本
- 已知限制：是否允许复位、是否允许写入、是否处于生产现场

## 3. 步骤 (Steps)
1. 确认电压、电平、接线与 `transport` 匹配。
2. 确认当前场景是否允许 `reset`、`halt` 或其他会改变现场的动作。
3. 优先使用无损连接与低风险频率，例如先降频再探测。
4. 若连接失败，先记录原始错误与尝试参数，再考虑更激进策略。
5. 在未明确授权前，不切换到破坏性动作，如 mass erase 或解锁流程。

## 4. 输出 (Output)
- `success`
- `current_state`：当前链路是否建立、目标是否可控
- `risk_or_limit`：当前未确认边界或禁止动作
- `raw_output`：OpenOCD 原始错误或握手日志

## 5. 风险 (Risks)
- 频率过高会导致链路不稳定或假性失联。
- 供电、电平或地线异常可能损坏调试器或目标板。
- 误判场景可能导致生产现场被复位或被改写。

## 6. 示例 (Example)
```text
结论：当前仅允许“读 + halt”，不允许写入。
原因：目标供电边界未确认，且用户未授权破坏性动作。
建议：使用 `adapter speed 1000` 进行只读探测连接。
```
