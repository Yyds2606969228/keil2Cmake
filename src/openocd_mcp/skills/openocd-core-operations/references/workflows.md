# 标准操作流模板

## 1. 目标 (Objective)
指导如何对目标执行结构化控制动作，确保每一步都有前提、验证和收尾。

## 2. 输入 (Input)
- 动作类型：读取、写入、刷写、校验、恢复
- 目标地址或对象
- 是否已获得高风险动作授权

## 3. 步骤 (Steps)
1. 前提：确认目标当前状态与动作边界匹配。
2. 冻结：如涉及写入或敏感读取，先 `halt`。
3. 执行：下发最小必要命令并设置超时。
4. 验证：对关键结果执行回读、比对或 `verify`。
5. 收尾：清理临时配置并声明最终状态。

## 4. 输出 (Output)
- 标准化执行结果
- 验证结果
- 错误回溯或下一步建议

## 5. 风险 (Risks)
- RDP、Option Bytes 或目标保护位会导致操作失败。
- 误写地址会破坏现场或业务逻辑。
- 未验证就宣告成功会扩大后续排查成本。

## 6. 示例 (Example)
```tcl
halt
flash write_image erase image.bin 0x08000000
verify_image image.bin 0x08000000
resume
```
