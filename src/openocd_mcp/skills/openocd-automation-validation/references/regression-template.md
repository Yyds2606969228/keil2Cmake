# 回归测试结果模板

## 1. 目标 (Objective)
规定自动化验证完成后统一提交的测试报告格式，确保结果可比较、可复盘、可交接。

## 2. 输入 (Input)
- 测试套件执行日志
- 预期断言基线
- 执行参数、次数、超时配置

## 3. 步骤 (Steps)
1. 统计通过、失败、超时、中止次数。
2. 汇总失败样本与首次失败轮次。
3. 对照基线，标明偏差是否在可接受范围内。
4. 如存在 flaky 行为，单独列出波动比例与样本条件。

## 4. 输出 (Output)
- Markdown 或 JSON 风格测试报告
- 失败样本摘要
- 是否建议进入深度分析

## 5. 风险 (Risks)
- 忽略 flaky 行为会产生虚假的“通过”。
- 只给通过率，不保留失败样本，会降低问题复现价值。
- 样本量过少时，统计结论不稳。

## 6. 示例 (Example)
```text
Total      : 100
Passed     : 97
Failed     : 2
Timed Out  : 1
First Fail : Iteration #45
Conclusion : 存在低概率连接超时，需要回放失败样本
```
