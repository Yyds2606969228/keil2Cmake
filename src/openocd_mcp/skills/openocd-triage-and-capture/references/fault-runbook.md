# 异常排查执行手册

## 1. 目标 (Objective)
提供应对 HardFault、BusFault、MemManage 等异常的标准急救排查步骤，用于先定类、再升级分析。

## 2. 输入 (Input)
- 目标已 `halt` 且现场尚未被重置
- `CFSR`、`HFSR`、`BFAR`、`MMFAR`、`SHCSR`
- 当前 PC、LR、SP

## 3. 步骤 (Steps)
1. 读取 Cortex-M 系统控制区相关寄存器。
2. 判断异常是否精确、是否可定位故障地址。
3. 记录 BFAR/MMFAR 并与 PC、SP 联动解释。
4. 若异常为非精确型，明确声明“当前 PC 可能不是根因点”。
5. 形成“异常类型 + 地址 + 置信度”的初判。

## 4. 输出 (Output)
- 初步异常定类
- 关键地址与寄存器清单
- 是否建议升级到深度分析

## 5. 风险 (Risks)
- 非精确错误无法直接定位真实出错指令。
- 现场已受中断或复位污染时，寄存器值可能不完整。
- 只看单个寄存器容易误判。

## 6. 示例 (Example)
```text
初判：精确总线错误 (PRECISERR)
地址：BFAR=0x40000000
依据：CFSR 对应位被置位，且 PC 停在外设访问指令附近
建议：进入深度分析，结合 ELF 确认具体语句
```
