# 证据

## 1. 用户目标
- keil2cmake 聚焦三项能力：
  - Keil `.uvprojx` -> CMake
  - CMake -> Keil 同步
  - OpenOCD 下载/调试模板配置
- `openocd_mcp` 与 `keil2cmake` 解耦，独立成模块（同仓库子项目）。

## 2. 官方资料
- CMake `set()` 语法（用于解析 `keil2cmake_user.cmake`）  
  链接: https://cmake.org/cmake/help/latest/command/set.html  
  关键结论: `set(VAR ...)` 支持列表和字符串参数，适合作为结构化同步输入。
- CMake Presets（用于保留 OpenOCD 模板的 configure 驱动能力）  
  链接: https://cmake.org/cmake/help/latest/manual/cmake-presets.7.html  
  关键结论: 可通过 preset cacheVariables 覆盖调试参数并触发配置阶段文件生成。
- OpenOCD 官方文档（接口/目标脚本模型）  
  链接: https://openocd.org/doc/html/index.html  
  关键结论: `interface/*.cfg` + `target/*.cfg` + transport 组合是标准配置模式。

## 3. 本地代码证据
- `src/keil2cmake/cli.py`
  - 原有 `mcp-debug` 子命令直接 `from openocd_mcp.server import main`，存在运行时耦合。
  - 已有 `openocd` 子命令仅生成模板文件，不依赖 MCP 服务。
- `src/keil2cmake/templates/keil2cmake_user.cmake.j2`
  - 已输出 `K2C_SOURCES/K2C_INCLUDE_DIRS/K2C_DEFINES/K2C_KEIL_MISC_*_FLAGS`，天然可作为 CMake->Keil 同步数据源。
- `src/keil2cmake/keil/uvprojx.py`
  - 现有读取路径明确了 Keil XML 的目标字段位置（IncludePath/Define/MiscControls/Groups）。
- `requirements.txt` 与 `Keil2Cmake.spec`
  - `fastmcp/pyserial/pyelftools/cmsis-svd/pyusb` 只为 `openocd_mcp` 服务，不属于 keil2cmake 核心依赖。

## 4. 现状判断
- 已确认：
  - keil2cmake 的 OpenOCD 模板能力与 mcp runtime 可分离。
  - `keil2cmake_user.cmake` 是稳定且可控的同步输入。
- 待确认：
  - `openocd_mcp` 子项目在不同运行入口（仓库根目录/子目录）的测试路径可用性。
- 风险点：
  - `.uvprojx` 不同模板的 XML 节点可能缺失，写回时需补齐节点层级。
  - `pyusb` 在无 backend 环境下可能抛出运行时异常，需容错。
