# ELF/符号表工作流

## 1. 目标 (Objective)
指导如何使用 ELF、符号表与地址映射工具，把裸地址还原为函数、源码位置与上下文。

## 2. 输入 (Input)
- 目标 `.elf` 文件
- 需解析的地址列表
- 当前运行镜像版本信息（如有）

## 3. 步骤 (Steps)
1. 确认 ELF 与板上镜像版本一致。
2. 检查 Section 布局与目标内存映射是否一致。
3. 使用符号工具将 PC、LR、返回地址映射到函数与源码行。
4. 若符号不完整，明确说明退化为“最近符号”推断。

## 4. 输出 (Output)
- 地址到函数/文件/行号的映射结果
- 符号匹配置信度
- 版本一致性说明

## 5. 风险 (Risks)
- ELF 与板载镜像不一致会导致完全错误的行号。
- 仅凭最近符号可能掩盖内联或优化带来的偏移。
- LTO 或高优化等级会降低源码映射直观性。

## 6. 示例 (Example)
```text
0x08001124 -> init_hardware() at src/main.c:42
0x080010F8 -> board_clock_init() at src/clock.c:18
说明：ELF 构建时间与板上镜像版本一致
```
