# 外设状态验证清单

## 1. 目标 (Objective)
提供常见外设的通用排错检查流，帮助从“没有信号”快速定位到“是哪一级配置有问题”。

## 2. 输入 (Input)
- 异常外设名
- 对应基地址或 SVD 节点
- 关联 GPIO、RCC、DMA 或中断信息

## 3. 步骤 (Steps)
1. 检查 RCC 时钟是否开启，复位是否被释放。
2. 检查 GPIO 模式、AF、上下拉、速度配置是否匹配。
3. 检查外设 CR/SR/ISR 等关键位是否已启用或报错。
4. 如涉及 DMA/IRQ，补充其使能与错误状态。

## 4. 输出 (Output)
- 外设检查结果清单
- 问题最可能出现的层级
- 下一步验证建议

## 5. 风险 (Risks)
- 忽视时钟树上游配置会得出错误结论。
- 引脚复用正确但波特率或采样时钟错误时，现象仍可能是“完全无输出”。
- 只看外设寄存器，不看 GPIO/中断/DMA，会遗漏关键问题。

## 6. 示例 (Example)
```text
外设：USART1
RCC：已开启
GPIO AF：正确
CR1.UE：0
初判：外设本体未启用，不是连线问题
```
