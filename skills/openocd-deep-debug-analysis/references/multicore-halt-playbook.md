# 多核同步挂起剧本

## 1. 目标 (Objective)
处理 AMP/SMP 多核架构下的联合调试场景，确保观察到的是一致的多核快照。

## 2. 输入 (Input)
- 多核 Target ID
- 主从核关系或共享资源关系
- 是否允许全核暂停

## 3. 步骤 (Steps)
1. 明确哪些核必须一起观察，哪些核允许单独处理。
2. 配置或模拟多核联动停止策略。
3. 对主核下发 `halt` 后，检查其余核是否已同步停下。
4. 分别记录每个核的 PC、SP、运行态与共享外设状态。

## 4. 输出 (Output)
- 多核快照表
- 是否同步冻结成功
- 潜在不同步风险说明

## 5. 风险 (Risks)
- 某一核未停下会继续改变共享状态。
- 外设和 DMA 仍在运行会破坏“静态现场”假设。
- Watchdog 可能在观察过程中重置系统。

## 6. 示例 (Example)
```text
Core0 (M7): halted, PC=0x08000000
Core1 (M4): halted, PC=0x24000123
Shared UART DMA: still active
结论：CPU 已冻结，但 DMA 仍可能污染缓冲区
```
