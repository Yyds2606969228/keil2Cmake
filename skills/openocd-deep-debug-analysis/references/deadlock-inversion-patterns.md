# 死锁与优先级反转模式

## 1. 目标 (Objective)
识别 RTOS 运行中常见的互斥量锁死、优先级反转与错误中断交互模型。

## 2. 输入 (Input)
- RTOS 任务控制块状态
- 互斥量或同步对象持有关系
- 当前中断嵌套层级

## 3. 步骤 (Steps)
1. 找出长期 `Blocked` 的高优先级任务。
2. 追踪其等待的锁、事件组或队列。
3. 确认资源持有者当前是否可被调度。
4. 检查是否存在中断里调用阻塞 API、锁顺序反转或优先级继承失效。

## 4. 输出 (Output)
- 死锁等待链
- 优先级反转证据
- 建议验证点

## 5. 风险 (Risks)
- 中断中调用阻塞 API 会掩盖真实锁链。
- 只看单个任务状态会漏掉资源持有者。
- TCB 已损坏时，等待链可能不完整。

## 6. 示例 (Example)
```text
Task A: Blocked on Lock1, priority=5
Task B: Ready, holds Lock1, priority=1
ISR X : 持续抢占，导致 Task B 无法释放资源
初判：优先级反转叠加抢占干扰
```
