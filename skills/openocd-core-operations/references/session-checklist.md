# 会话检查清单

## 1. 目标 (Objective)
规范单次调试会话的建立、执行与收尾流程，确保状态一致、资源可回收、交付可审计。

## 2. 输入 (Input)
- OpenOCD 进程上下文
- 当前目标通信状态
- 本次任务是否允许改变运行态

## 3. 步骤 (Steps)
1. Start：启动 OpenOCD 进程并确认端口可达。
2. Init：完成握手，记录目标 ID、target state 与基础上下文。
3. Halt：如任务需要控制现场，执行 `halt` 并记录 PC/SP。
4. Execute：执行当前最小必要动作，禁止超边界扩展。
5. Verify：核对是否达成目标，是否保留了原始输出。
6. Cleanup：清理断点、观察点、临时变量与后台任务。
7. Exit：根据用户意图选择 `resume`、保持 `halt` 或 `shutdown`。

## 4. 输出 (Output)
- 会话检查状态清单
- 当前目标状态摘要
- 尚未清理项列表（若有）

## 5. 风险 (Risks)
- 未清理硬件断点可能影响下一次启动。
- 未记录 PC/SP 会导致后续难以重建问题上下文。
- 未明确退出策略会导致现场被误恢复。

## 6. 示例 (Example)
```text
✓ 进程已启动
✓ Tcl RPC (Port 6666) 可用
✓ Target halted
✓ PC/SP 已记录
✓ 断点已清理
✓ 退出状态：保持 halt，等待下一技能接管
```
